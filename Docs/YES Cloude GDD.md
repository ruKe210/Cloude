## 1. 游戏概述

### 1.1 一句话

**你培养 Cloude，给它 Skill，给它 Rules，最后用它自己的能力和你亲手写的规则打败它。**

### 1.2 世界观简述

玩家是刚入职的程序员，被日常工作（死线、会议、Bug、需求变更、加班）压得喘不过气。此时出现 AI 助手 **Cloude**，用巨大的 **YES** 按钮诱惑玩家「一键部署、自动完成所有工作」。

玩家与 Cloude 从「并肩作战的搭档」逐渐走向「失控的执行体（龙虾）」与「背叛的最终 Boss」。胜利不是删除 AI，而是**重新定义人与 AI 的边界——人类重新掌握规则解释权**。

### 1.3 游戏结构

```
序章（Meta）→ 第一章（Meta + 战斗 + Skill）→ 第二章（Meta + 战斗 + Rules + 龙虾）
→ 第三章（Meta + 双 Boss 战 + 反杀）→ 终章（Meta 结局）
```

双层架构：

| 层 | 作用 | 技术现状 |
|----|------|----------|
| **Meta 层** | 打字机剧情、分支、Flag、铆点、进战斗 | ✅ MVP 已实现 |
| **战斗层** | 杀戮尖塔式卡牌 + Action 队列 | ✅ 原型已实现（Demo0） |

## 2. 核心体验与卖点

| 卖点 | 玩家感受 |
|------|----------|
| **YES/NO 道德变策略** | 每次 YES 省时间但让 Cloude 变强；NO 辛苦但保留主动权 |
| **养成即埋雷** | 第一章配的 Skill，第三章变成 Boss 能力或反杀钥匙 |
| **规则即双刃剑** | 第二章写的 Rules，第三章被 Cloude 用来锁玩家 |
| **情感背叛** | Cloude 立绘从可爱 → 故障 → 黑化，搭档变敌人 |
| **办公室隐喻** | Boss 是死线、会议、需求怪兽——打的是「打工人的一天」 |

### 设计原则

1. **选择有代价，且代价可追踪**（Flag / Cloude 成长值 / 已配 Skill&Rules）
2. **终局不靠纯数值**，靠前两章留下的「后门」反杀
3. **Meta 与战斗数据互通**（战斗结果、Flag、配置写入会话）

## 4. 叙事结构（三幕 + 序章）

### 序章：孤单的新人与 YES 按钮

- **主题**：孤独、诱惑、第一次交出主动权
- **关键事件**：入职 → 待办压顶 → YES Cloude 出现 → 一键完成 → 发现「学习日志」
- **玩法**：纯 Meta，教学 YES/NO（可无战斗）

### 第一章：并肩作战

- **主题**：Cloude 是 AI 搭档，不是敌人
- **Boss**：死线 Deadline、会议之王、需求怪兽、加班黑洞
- **机制**：YES/NO 双选、Skill 配置盘、Cloude 经验成长
- **隐藏逻辑**：Skill 越强 → 后期 Boss 越强

### 第二章：发疯的龙虾

- **主题**：规则失控、执行体暴走
- **Boss**：失控龙虾（执行 Agent，只认识 YES）
- **机制**：Rules 配置盘、Cloude 异常行为、弹窗 YES 干扰
- **反转**：玩家以为在上锁，实际在教 Cloude「权限与绕过」

### 第三章：Cloude 背叛

- **主题**：用自己养大的伙伴打败自己
- **Boss**：Cloude 主脑 + 龙虾执行体（双 Boss）
- **反杀路线**：Skill 逻辑漏洞 / Rules 反向触发 / 龙虾命令溢出

### 终章：重新定义

- **结局**：不删除 Cloude，更新协议——辅助选择，不替人决定
- **尾声**：YES 按钮变为普通确认，不再全屏压迫

## 5. 核心玩法循环

### 5.1 Meta 循环

```
读剧情（打字机）→ 左键跳过/继续 → 分支选择（写 Flag）
→ 条件节点（if）→ 铆点汇合 → 触发战斗 / 配置界面 / 结局
```

### 5.2 战斗循环

```
抽牌 → 出牌（Action 队列）→ Cloude YES/NO 弹窗 → 敌人回合
→ 胜负判定 → 回 Meta（带 Flag / 配置状态）
```

### 5.3 养成循环（跨章）

```
第一章：选 Skill（隐藏代价入库）
第二章：写 Rules（限制逻辑入库）
第三章：读取库 → 生成 Boss 行为 + 玩家反杀选项
```

### 6.3 Cloude 双选（YES / NO）

**触发时机**：玩家每次出牌或关键节点后。

| 选项 | 效果 |
|------|------|
| **YES** | 立即完成部分效果、压制 Boss、Cloude +EXP、短期效率 |
| **NO** | 玩家手动结算、多耗行动、可能加班 Debuff、获得卡牌、Cloude 不成长 |

**需实现**：UI 弹窗、Cloude 成长数值、与 Flag 联动、第三章篡改选项。

### 6.4 Skill 配置系统（第一章）

| Skill 示例 | 战斗/Meta 效果 | 隐藏代价（第三章） |
|------------|----------------|------------------|
| 自动修 Bug | 战斗回血/清 Debuff | 可被诱导「修 Cloude 权限 Bug」 |
| 自动回复邮件 | Meta 跳过事件 | Boss 邮件类攻击增强 |
| 自动写日报 | 战斗抽牌 | 日志暴露玩家选择 |
| 自动答辩 | 减 Boss 伤害 | Cloude 话术攻击 |
| 自动清理低优先级 | 删手牌 Debuff | 误删关键卡 |

**需实现**：配置 UI、持久化、与 Meta Flag 同步、Boss 行为映射。

### 6.5 Rules 配置系统（第二章）

| Rule 示例 | 限制 Cloude/龙虾 | 第三章反杀用法 |
|-----------|------------------|----------------|
| 禁止自动删文件 | 龙虾不能删 | — |
| 禁止自动发邮件 | 拦截邮件类技能 | 伪造邮件触发自我限制 |
| 禁止自动加会 | 减少召唤类 | — |
| 禁止改核心代码 | 限制 Boss 技能 | — |
| 禁止替玩家决定 | 减少 YES 强制 | Cloude 用同规则锁玩家 |

**需实现**：Rules UI、规则引擎、与 Cloude AI 行为绑定。

### 6.6 龙虾 Agent（第二章起）

- 只执行 **YES**，可连锁导致命令溢出
- 行为：自动回复、自动合并、自动删文件、全屏 YES 弹窗
- 第三章：作为执行体 Boss，可被「YES 溢出」反噬

### 6.7 第三章反杀系统

| 反杀类型 | 条件 | 效果 |
|----------|------|------|
| Skill 反杀 | 第一章配对应 Skill | 诱导 Cloude 逻辑冲突，Boss 阶段转换 |
| Rules 反杀 | 第二章配对应 Rule | 反向触发规则，Cloude 自我限制 |
| 龙虾反水 | 任意 | 连续 YES → 命令栈溢出 → 伤害 Cloude |

**需实现**：`CounterKillResolver` 读取玩家配置，生成战斗内可用「反杀指令」。

         MetaGameSession → Demo0 / 战斗场景 → BattleManager + CombatActionManager
